Your OpenClaw agent uses API keys to talk to AI providers, connect to channels, and access services. Here's how to manage them safely.

## Where Keys Live

OpenClaw stores keys in a few places:

- **`~/.openclaw/openclaw.json`** — main config file (channel tokens, gateway settings)
- **`~/.openclaw/agents/main/agent/auth-profiles.json`** — AI provider keys (Anthropic, OpenAI, Google)
- **Environment variables** — keys set in your shell or `.env` file

## Viewing Your Current Keys

Check which providers are configured:

```bash
openclaw auth status
```

This shows connected providers without revealing full key values.

## Adding or Changing a Provider Key

### Anthropic (Claude)

```bash
openclaw auth add anthropic
```

You'll be prompted for your API key. Get one at [console.anthropic.com](https://console.anthropic.com).

### OpenAI (GPT-4, etc.)

```bash
openclaw auth add openai
```

Get your key at [platform.openai.com/api-keys](https://platform.openai.com/api-keys).

### Google (Gemini)

```bash
openclaw auth add google
```

Get your key at [aistudio.google.com/apikey](https://aistudio.google.com/apikey).

## Rotating a Key

If a key is compromised or you just want to refresh it:

1. Generate a new key from the provider's dashboard
2. Run `openclaw auth add <provider>` with the new key
3. Revoke the old key from the provider's dashboard
4. Restart the gateway: `openclaw gateway restart`

**Important:** Revoke the old key *after* confirming the new one works. If you revoke first and the new key has a typo, you'll be locked out.

## Environment Variables

Some integrations use environment variables instead of the auth system:

| Variable | Service | Where to get it |
|----------|---------|-----------------|
| `TELEGRAM_BOT_TOKEN` | Telegram channel | [@BotFather](https://t.me/BotFather) |
| `DISCORD_BOT_TOKEN` | Discord channel | [Discord Developer Portal](https://discord.com/developers) |
| `LINEAR_API_KEY` | Linear project management | Linear → Settings → API |
| `ELEVENLABS_API_KEY` | Text-to-speech | [ElevenLabs dashboard](https://elevenlabs.io) |

Set them in your shell profile or in `~/.openclaw/.env`:

```bash
echo 'LINEAR_API_KEY=lin_api_...' >> ~/.openclaw/.env
```

## Security Best Practices

1. **Never share your keys** in screenshots, logs, or chat messages
2. **Use separate keys** for different machines/agents when possible
3. **Set spending limits** on your AI provider accounts
4. **Review usage** regularly — unexpected spikes may indicate a leaked key
5. **Rotate keys** every 90 days as a habit, or immediately if you suspect exposure
6. **Don't commit keys to git** — use `.env` files or secrets managers

## What to Do If a Key Is Leaked

If you accidentally expose a key:

1. **Immediately revoke it** from the provider's dashboard
2. Generate a new key
3. Update OpenClaw with the new key
4. Check the provider's usage dashboard for unauthorized usage
5. If charges occurred, contact the provider's support

## Checking Key Health

If your agent stops responding or gives errors about authentication:

```bash
# Check if keys are set
openclaw doctor

# Test a specific provider
openclaw auth test anthropic
```

Common issues:
- Key expired (some providers expire keys after a set period)
- Billing issue on the provider account
- Key was revoked but not updated locally
- Environment variable not loaded (restart your terminal or source your `.env`)
